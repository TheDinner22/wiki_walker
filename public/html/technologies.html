Technologies Used:

HTMX

HTMX is the best front-end framework in the world! Fast and easy to use. HTMX made it easier to interface with an extremely low level (C++) backend. We can respond to requests with html, json, or both! We can have requests update the DOM or trigger custom JS functions.

LibCurl

We needed to interact with Wikipedia using HTTP requests. We needed to be able to send http requests in C++! This meant using some external library to send requests for us. LibCurl is free and relatively easy to use (not to install though). It also handles other miscellaneous web related things such as URL encoding for us.

SimdJSON

The Wikipedia API we are using returns JSON. Handling JSON is a very common requirement and it wasn’t hard to find a JSON parser written in and for C++. SimdJSON is fast and extremely easy to install (single header library). It's worth noting that SimdJSON only supports JSON decoding; Anytime we needed to encode some data into JSON, it was done manually (with string literals and lots of concatenations.

HTTPRequest

I’m truly shocked I found something as good as HTTPRequest. Setting up a web 1.0 REST-ful backend in CPP was extremely easy. This single header file library supports file based routing (for our public folder). It also supports redirects, custom routing, and https. GO STAR IT ON GITHUB.

Docker

We developed and deployed our code with Docker! Deploying with Docker was easy. Using Docker while developing meant only installing everything once. See Example Use for a video tutorial on Docker.
